## Overview

The widget ships with four built-in pages:

- `HOME`
- `ARTICLES`
- `CONVERSATION`
- `CONVERSATION_HISTORY`

You can extend or replace them in two ways:

- `customPages` for the default router
- `Support.Router` plus `Support.Page` for full composition

## Replace a Built-In Page with `customPages`

If you want to keep the default widget shell and router, pass a page override
directly to `Support` or `Support.Root`.

```tsx
import { Support, useSupportNavigation } from "@cossistant/react";

function CustomHomePage() {
  const { navigate } = useSupportNavigation();

  return (
    <div className="flex h-full flex-col p-6">
      <h2 className="text-2xl font-semibold">Launch support</h2>
      <p className="mt-2 text-sm text-zinc-500">
        Swap the default home page without rebuilding the conversation flow.
      </p>

      <button
        className="mt-6 rounded-xl border px-4 py-3 text-left"
        onClick={() =>
          navigate({
            page: "CONVERSATION",
            params: {
              conversationId: "pending_docs_conversation",
              initialMessage: "Help me launch with the React SDK",
            },
          })
        }
        type="button"
      >
        Start a conversation
      </button>
    </div>
  );
}

<Support
  customPages={[
    {
      name: "HOME",
      component: CustomHomePage,
    },
  ]}
/>;
```

`customPages` beats the built-in page. If you also pass `slots.homePage`, the
explicit route override still wins.

## Use `Support.Page` with the Default Widget

Top-level `Support.Page` registration works with `Support`, even when you also
render `Support.Content` for custom positioning or styling.

```tsx
import { Support, useSupportNavigation } from "@cossistant/react";

function FAQPage() {
  const { goBack } = useSupportNavigation();

  return (
    <div className="p-6">
      <button onClick={goBack} type="button">
        Back
      </button>
      <h2 className="mt-4 text-xl font-semibold">FAQ</h2>
    </div>
  );
}

<Support open>
  <Support.Content className="rounded-3xl border shadow-2xl" />
  <Support.Page component={FAQPage} name="FAQ" />
</Support>;
```

## Full Composition with `Support.Root`

Use `Support.Root` when you want your own shell and explicit router tree.

<ComponentPreview name="support-full-composition" />

```tsx
import { Support } from "@cossistant/react";

function LaunchChecklistPage() {
  return <div className="p-6">Your custom page</div>;
}

export function App() {
  return (
    <Support.Root open>
      <Support.Trigger asChild>
        <button type="button">Compose support</button>
      </Support.Trigger>

      <Support.Content className="rounded-3xl border shadow-2xl">
        <Support.Router>
          <Support.Page component={LaunchChecklistPage} name="HOME" />
        </Support.Router>
      </Support.Content>
    </Support.Root>
  );
}
```

With `Support.Root`, place custom pages inside `Support.Router` or pass
`customPages` to `Support.Root` when you want the default router inside
`Support.Content`.

## Navigation Hook

Use `useSupportNavigation()` anywhere inside the widget tree.

```tsx
import { useSupportNavigation } from "@cossistant/react";

function NavigationButtons() {
  const { canGoBack, goBack, navigate, page, replace } = useSupportNavigation();

  return (
    <div className="flex gap-2">
      {canGoBack && (
        <button onClick={goBack} type="button">
          Back
        </button>
      )}

      <button onClick={() => navigate({ page: "HOME" })} type="button">
        Home
      </button>

      <button onClick={() => replace({ page: "ARTICLES" })} type="button">
        Articles
      </button>

      <span>Current page: {page}</span>
    </div>
  );
}
```

## Type Augmentation for Custom Routes

If you add custom routes, you can extend the route registry for full TypeScript
autocomplete.

```tsx
declare module "@cossistant/core" {
  interface RouteRegistry {
    FAQ: undefined;
    SETTINGS: { tab: string };
  }
}
```

Then your navigation calls become type-safe:

```tsx
navigate({
  page: "SETTINGS",
  params: { tab: "profile" },
});
```
