## Philosophy

The React support SDK has two lanes on purpose:

- `Support` is the fastest way to ship a ready-to-use support widget.
- `slots` and `slotProps` are the fastest way to swap one part without rebuilding everything.
- `Support.Root` plus `Support.*` is the full composition lane when you want your own shell.
- `Primitives` are the advanced headless building blocks.

Start with the default widget. Move into composition only when you actually need it.

## Quick Start

```tsx
import { Support, SupportProvider } from "@cossistant/react";
import "@cossistant/react/styles.css";

export function App() {
  return (
    <SupportProvider publicKey="pk_live_...">
      <Support />
    </SupportProvider>
  );
}
```

The widget does not inject CSS automatically. Use one of these imports at your
app root:

- `@cossistant/react/styles.css` for plain CSS or any non-Tailwind app
- `@cossistant/react/support.css` if your app already uses Tailwind CSS v4

## Quick Customization

The easiest way to customize the widget is to keep the default runtime and
replace only the parts you care about.

```tsx
import { Support } from "@cossistant/react";

<Support
  slots={{
    trigger: CustomBubble,
    homePage: CustomHomePage,
  }}
  slotProps={{
    content: {
      className: "rounded-3xl border border-neutral-200 shadow-2xl",
    },
  }}
/>;
```

Use `slots` when you want a new component. Use `slotProps` when you want to add
presentational props like `className` without replacing the whole part.

## Visitor Identity and Per-Page Defaults

Use `IdentifySupportVisitor` to connect the widget to your signed-in user, and
use `SupportConfig` when you want page-specific quick options or welcome
messages.

```tsx title="app/pricing/page.tsx"
import {
  IdentifySupportVisitor,
  SupportConfig,
} from "@cossistant/react";
import { SenderType, type DefaultMessage } from "@cossistant/types";

const defaultMessages: DefaultMessage[] = [
  {
    content: "Welcome to pricing. I can help with plans, billing, or migration.",
    senderType: SenderType.TEAM_MEMBER,
  },
];

export function PricingPage({
  user,
}: {
  user: {
    id: string;
    email: string;
    name: string | null;
    image?: string | null;
  };
}) {
  return (
    <>
      <IdentifySupportVisitor
        email={user.email}
        externalId={user.id}
        image={user.image}
        name={user.name}
      />

      <SupportConfig
        defaultMessages={defaultMessages}
        quickOptions={[
          "How does billing work?",
          "Can I cancel anytime?",
          "What is included in Pro?",
        ]}
      />
    </>
  );
}
```

## Choosing the Right API

- Use `Support` when you want the ready-made widget.
- Use `Support` with `slots` when you want to swap the trigger, home page,
  composer, timeline, or another built-in part.
- Use `Support.Root`, `Support.Trigger`, `Support.Content`, `Support.Router`,
  and `Support.Page` when you want full composition.
- Use `Primitives` when you want a headless build.
