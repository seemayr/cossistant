## Overview

The recommended customization path is:

1. Keep `Support` for the runtime and defaults.
2. Use `classNames` and `slotProps` for styling.
3. Use `slots` to replace only the parts you need.
4. Move to `Support.Root` only when you want a custom shell.

## Styling Basics

Import one stylesheet at your app root:

```tsx
import "@cossistant/react/styles.css";
```

Or, if your app already uses Tailwind CSS v4:

```tsx
import "@cossistant/react/support.css";
```

For quick styling changes, start with `classNames` and `slotProps`:

```tsx
<Support
  classNames={{
    trigger: "bg-black text-white",
    content: "border-neutral-200 shadow-2xl",
  }}
  slotProps={{
    content: {
      className: "rounded-[28px]",
    },
  }}
/>
```

If you want broader theming without replacing components, override the theme
tokens exposed by the stylesheet:

```css
:root {
  --co-theme-radius: 1.5rem;
  --co-theme-background: oklch(99% 0.01 95);
  --co-theme-foreground: oklch(18% 0.01 95);
  --co-theme-primary: oklch(22% 0.03 250);
  --co-theme-primary-foreground: white;
  --co-theme-border: oklch(90% 0.01 95);
}
```

### Stable styling hooks

The default widget exposes stable DOM attributes for CSS and Tailwind selectors:

- `data-slot`
- `data-state`
- `data-page`

```css
[data-slot="trigger"][data-state="open"] {
  transform: scale(1.02);
}

[data-slot="content"] {
  backdrop-filter: blur(12px);
}
```
